Build a Student Management System as a mobile-first application with three main roles: Student, Staff, and Admin. 
The app should be developed using the following tech stack:
- Frontend: HTML, CSS, JavaScript (if needed), wrapped in a mobile WebView or PWA for mobile deployment.
- Backend: Java with Spring Boot.
- Database: MySQL.

### STUDENT MODULE
- Authentication:
  - Sign up and log in using Student Identity Number + Phone Number with OTP verification.
- Features:
  - Dashboard showing current semester subjects.
  - Tap on a subject → open study documents (PDF, notes, slides).
  - Attendance via QR code scanning:
    - QR code generated by staff.
    - QR must refresh/change every 10 seconds for security.
  - Fee payment:
    - Redirect to external payment apps (GPay, PhonePay).
    - Ensure smooth mobile redirection and return flow.

### STAFF MODULE
- Authentication:
  - Staff login with Staff ID + Password (provided by Admin).
- Staff Types:
  1. **Class Incharge**
     - Can edit/update student details.
     - Responsible for generating student passwords.
  2. **Subject Incharge**
     - Can only enter test marks.
     - No permission to modify student details.
- Role-based restrictions must be enforced at both mobile UI and backend API levels.

### ADMIN MODULE
- Authentication:
  - Admin login with Admin ID + Password.
- Features:
  - Full control over the system.
  - Create/manage staff accounts.
  - Generate and assign staff passwords.
  - Manage students, staff, subjects, fees, attendance, and documents.

### SYSTEM REQUIREMENTS
- Mobile-first design:
  - Responsive UI with dashboards for each role.
  - Native-like experience using WebView/PWA.
- Security:
  - OTP-based login for students.
  - Role-based authentication and authorization.
  - QR attendance system with dynamic refresh every 10 seconds.
- Integrations:
  - Payment gateway redirection (GPay, PhonePay).
  - Document management (upload, view, download).
- Backend:
  - REST APIs (Spring Boot).
  - JWT-based authentication.
- Database:
  - MySQL schema with tables for students, staff, admin, subjects, documents, attendance, marks, fees, semesters.
- Architecture:
  - Clean architecture with separation of concerns.
  - Scalable and maintainable codebase.

### OUTPUT EXPECTATION
- Provide:
  - Backend API design (authentication, subjects, attendance, fees, staff management).
  - Database schema (students, staff, admin, subjects, attendance, fees).
  - Mobile UI/UX flow (login, dashboard, subject view, attendance scan, fee payment).
  - Role-based access control implementation.
