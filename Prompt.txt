# STUDENT MODULE

## Authentication

Login using:

* Student Identity Number(Roll No) unique for all student which given by staff
* Phone Number

Flow:

1. Generate 6-digit OTP
2. Send via SMS/WhatsApp gateway
3. Store OTP log
4. Validate OTP
5. Issue JWT
6. Session valid for 7 days
7. Allow logout all devices

Implement rate limiting and brute-force protection.

## QR Attendance System

Staff generates QR session.

QR Requirements:

* Rotates every 10 seconds
* Contains session ID + timestamp + encrypted signature
* Auto expires

Student scanner:

* Opens camera
* Reads QR
* Captures GPS
* Sends to backend

Backend validates:

* QR freshness (every 10 seconds)
* Session active
* Student enrolled
* GPS within allowed radius
* Prevent duplicate attendance

On success:

* Trigger vibration
* Show green check animation

Staff dashboard shows live success count.
